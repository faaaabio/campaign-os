The canonical agent prompt for v1 lives at `agents/cgi.md`. This file is a pointer.

## Why this is just a pointer

The first version of every agent prompt IS the agent file in `agents/`. Retros produce deltas — `v2.md`, `v3.md`, etc. — that override or extend `v1`.

When the Analyst reads "prompt_version: v1" in a render's metadata, they're attributing performance to the unedited canonical agent file at the time of that sprint.

## Override pattern

Starting with v2, retros may produce files like:

```
prompts/cgi/v2.md
```

Each new version is a complete prompt (not just a diff), produced by the agent itself after reading the retro's recommendations. The latest `v{n}.md` is what loads on `/sprint-start`.

If you're reading this in a fresh repo, you're on v1 — load `agents/cgi.md` and proceed.
